## AI-Generated Contributions Policy

This project does **not** accept pull requests or code contributions that are wholly or substantially generated by AI tools (including, but not limited to, large language models, code-generation assistants, and automated coding agents).

### Why?

#### 1. Educational Purpose

One of the primary goals of this project is education.

Contributors are encouraged to learn about game engines, RTS architecture, graphics programming, networking, pathfinding, AI, and software engineering by actively designing and implementing solutions themselves. Submitting AI-generated code bypasses much of the learning process and works against this objective.

#### 2. Reviewer Time Is Limited

Reviewing and maintaining code is often more expensive than writing it.

AI-generated pull requests frequently contain large amounts of code that appear plausible at first glance but require extensive verification for correctness, maintainability, performance, and architectural consistency. The effort required to review such contributions often exceeds the value they provide.

Therefore, reviewing AI generated code is not the best way to spend reviewers' time.

#### 3. Long-Term Code Quality

This project prioritizes correctness, readability, maintainability, and architectural consistency over rapid code generation. Even when the generated code appears functional, it frequently increases long-term maintenance and troublshooting costs.

#### 4. Understanding Matters More Than Output

Contributors should be able to explain:

* Why a solution was chosen
* How the implementation works
* What trade-offs were considered
* How the code interacts with the rest of the game

Code that cannot be confidently explained and defended by its author is unlikely to be accepted.

#### 5. Preserving Project Authenticity

This project aims to be a community effort built through genuine engineering work, experimentation, problem-solving, and learning. The value of the project comes not only from the final codebase, but also from the knowledge gained while building it.

### Allowed Uses

AI tools may be used for:

* Grammar and spelling assistance
* Research and reference gathering
* Small-scale experimentation that is not submitted directly as project code
